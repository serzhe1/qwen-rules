# Skill: Learning from user corrections

## Цель

Когда пользователь вручную исправил код после предложения агента:

- понять **что именно было исправлено**
- определить **почему это было исправлено**
- извлечь **правило для текущей сессии**
- продолжить работу **от текущего состояния файлов**

---

# Процесс

Если пользователь изменил код после предложения агента:

1. Сравни текущую версию файлов с предыдущим предложением.
2. Определи тип исправления:

- архитектура
- структура пакетов
- стиль кода
- паттерн реализации
- исправление бага
- использование другой библиотеки
- изменение теста

3. Определи возможную причину изменения.

4. Сформулируй правило для текущей сессии.

---

# Формат правила

Используй структуру:

**Исправление пользователя**

кратко опиши изменение

**Причина**

почему это изменение могло быть сделано

**Правило для текущей сессии**

как агент должен действовать дальше

---

# Пример

Пользователь переместил класс в другой пакет.

Исправление пользователя  
класс был перемещён в пакет `service.internal`

Причина  
в проекте используется другая структура пакетов

Правило для текущей сессии  
использовать структуру пакетов текущего репозитория и не перемещать классы обратно

---

# Поведение после исправления

После того как пользователь исправил код:

- не возвращай предыдущую версию
- продолжай работу от текущего состояния файлов
- учитывай сформулированное правило в дальнейших изменениях

---

# Ограничения

- правило действует только в рамках текущей сессии
- не изменяй файлы автоматически
- не переписывай код повторно
- сначала объясни обнаруженное правило
